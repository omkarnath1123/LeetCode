You have a convex `n`-sided polygon where each vertex has an integer value. You are given an integer array `values` where `values[i]` is the value of the `ith` vertex (i.e., **clockwise order**).

You will **triangulate** the polygon into `n - 2` triangles. For each triangle, the value of that triangle is the product of the values of its vertices, and the total score of the triangulation is the sum of these values over all `n - 2` triangles in the triangulation.

Return _the smallest possible total score that you can achieve with some triangulation of the polygon_.

**Example 1:**

![](https://assets.leetcode.com/uploads/2021/02/25/shape1.jpg)
```
Input: values = [1,2,3]
Output: 6
Explanation: The polygon is already triangulated, and the score of the only triangle is 6.
```
**Example 2:**

![](https://assets.leetcode.com/uploads/2021/02/25/shape2.jpg)
```
Input: values = [3,7,4,5]
Output: 144
Explanation: There are two triangulations, with possible scores: 3*7*5 + 4*5*7 = 245, or 3*4*5 + 3*4*7 = 144.
The minimum score is 144.
```
**Example 3:**

![](https://assets.leetcode.com/uploads/2021/02/25/shape3.jpg)
```
Input: values = [1,3,1,4,1,5]
Output: 13
Explanation: The minimum score triangulation has score 1*1*3 + 1*1*4 + 1*1*5 + 1*1*1 = 13.
```
**Constraints:**

-   `n == values.length`
-   `3 <= n <= 50`
-   `1 <= values[i] <= 100`

## Detailed Intuition

### 💡 How to Think About the Solution

1. **Understanding Triangulation:**
   - A triangulation divides a polygon into triangles using non-intersecting diagonals
   - For an n-sided polygon, we need exactly (n-2) triangles
   - Each triangle's score is the product of its vertex values
   - We want to minimize the sum of all triangle scores

2. **Key Insights:**
   - We can break this down into subproblems
   - Any diagonal divides the polygon into two parts
   - Each part can be triangulated independently
   - The final answer builds up from smaller polygons

### 🔍 Edge Cases to Consider
- Minimum polygon (3 vertices)
- Maximum polygon (50 vertices)
- All values being the same
- Large differences between adjacent values
- Values near the constraints (1 to 100)

### 🎯 Solution Approaches

1. **Dynamic Programming Approach**
   ```pseudo
   // dp[i][j] represents minimum score for polygon from vertex i to j
   for len from 3 to n:
       for i from 0 to n-len:
           j = i + len - 1
           dp[i][j] = min(dp[i][k] + dp[k][j] + values[i]*values[k]*values[j])
                      for all k from i+1 to j-1
   ```
   Time: O(n³), Space: O(n²)

### 📈 Visual Explanation
```
For polygon [3,7,4,5]:

Step 1: Base triangles (size 3)
[3,7,4]: min score = 3*7*4 = 84
[7,4,5]: min score = 7*4*5 = 140
...

Step 2: Full polygon
Choice 1: (3,7,5) + (3,4,5) = 105 + 60 = 165
Choice 2: (3,4,5) + (3,4,7) = 60 + 84 = 144 ← Minimum!
```

1. Use 2D DP table to store minimum scores
2. Process subpolygons from smaller to larger sizes
3. Consider all possible ways to split each subpolygon
4. Initialize base cases properly (triangles of size 3)

## Detailed Intuition

- Define DP state so each state captures the minimum information required to continue transitions.
- Write transition from previously solved states and compute in valid order.
- Initialize base cases carefully since all later states depend on them.

```java
    Integer[][] store;
    int len;
    public int minScoreTriangulation(int[] values) {
        store = new Integer[50][50];
        len = values.length;
        
        return backTrack(0, len - 1, values);
    }
    private int backTrack(int i, int j, int[] values) {
        if (j <= i + 1) return 0;// need three points to create the triangle 
        if (store[i][j] != null)
            return store[i][j];
        
        int min = Integer.MAX_VALUE;
        for (int k = i + 1; k < j; k++)
            min = Math.min(min, backTrack(i, k, values) + 
                                backTrack(k, j, values) + 
                                values[i] * values[j] * values[k]);
        store[i][j] = min;
        return min;
    }
```
