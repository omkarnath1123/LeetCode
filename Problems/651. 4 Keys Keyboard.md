Imagine you have a special keyboard with the following keys:

-   A: Print one `'A'` on the screen.
-   Ctrl-A: Select the whole screen.
-   Ctrl-C: Copy selection to buffer.
-   Ctrl-V: Print buffer on screen appending it after what has already been printed.

Given an integer n, return _the maximum number of_ `'A'` _you can print on the screen with **at most**_ `n` _presses on the keys_.

**Example 1:**
```
Input: n = 3
Output: 3
Explanation: We can at most get 3 A's on screen by pressing the following key sequence:
A, A, A
```
**Example 2:**
```
Input: n = 7
Output: 9
Explanation: We can at most get 9 A's on screen by pressing following key sequence:
A, A, A, Ctrl A, Ctrl C, Ctrl V, Ctrl V
```
**Constraints:**
-   `1 <= n <= 50`

**Solution:**

- [x] [[Utils/Dynamic Programming]]

## Detailed Intuition

- Define DP state so each state captures the minimum information required to continue transitions.
- Write transition from previously solved states and compute in valid order.
- Initialize base cases carefully since all later states depend on them.

**Time Complexity** : O(n)<br>
**Space Complexity** : O(n)

```java
    Map<Integer, Integer> store = new HashMap<>();
    public int maxA(int n) {
        return backTrack(n);
    }
    private int backTrack(int n) {
        if (n <= 3) return n;
        if (store.containsKey(n))
            return store.get(n);
        
        int max = n;
        for (int i = 2; i <= n - 3; i++)
            max = Math.max(max, (n - i - 1) * backTrack(i));
        
        store.put(n, max);
        return max;
    }
```
